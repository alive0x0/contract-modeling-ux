Phase 1 v2 — Changes from Feedback

My Scenarios / All Scenarios NEW
Tab toggle at the top of the table. "My Scenarios" scoped to the current user's 24 scenarios by default — avoids drowning in all 312. Switching to "All" shows the org-wide set. Count badges update live with filters applied.
Advanced Filters panel NEW
Opened via the "Advanced Filters" button. Contains: Region, Contract Office, System Name, CRP Type, LOB, Created By, Date Created range, Last Run range, Provider Count range, and Tag. Quick filter bar stays lean — CRP Type moved here.
CRP Type — moved to Advanced Filters MOVED
Removed from the quick filter bar per feedback. Still fully filterable in Advanced Filters. The quick bar now only carries Claim Type, Run Status, RSH Pushed, and scenario type pills — the four most frequent daily lookups.
Pushed to RSH — Yes / No SIMPLIFIED
Replaced the dot indicator (Synced/Stale/Unknown) with a plain Yes/No badge. Matches how users actually think about it from their Excel tracker. The live API feed still powers it — but the surface is binary.
System Name column NEW
Shows which contracting system the scenario belongs to (e.g. Medicare Advantage, Commercial PPO, ASC Network). Filterable in Advanced Filters. Addresses the need to distinguish scenarios across institutional systems without opening each one.
Region column NEW
Geographic region of the contract office (South Central, Pacific NW, etc.). Distinct from Contract Office — users often filter by region first, then office. Filterable in Advanced Filters. Both columns visible by default.
Provider Count column NEW
Number of providers in scope for this scenario. Hover shows tooltip. Filterable in Advanced Filters by min/max range. Addresses the need to quickly gauge scenario scale without opening the scenario detail page.
Date Created + Last Run with timestamps ENHANCED
Both date columns now show date + time. Date Created is separate from Last Run — users care about both: when did I first build this, and when did I last execute it. Sortable independently.
